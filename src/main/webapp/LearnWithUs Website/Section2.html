OPERATOR

This are the symbol which perform some specific task.


  • TYPE OF OPERATOR:
  • Arithmetic Operator:

    • The arithmetic operators perform addition(+), subtraction(-), multiplication(*), division(/), exponentiation(^) operation.


  • Concatination Operator:

    • The concatenation operator is a binary operator. You can use the concatenation operator ( || ) to concatenate two expressions that evaluate to character data types or to numeric data types.


  • Comparision Operator:

    • Checks if the values of two operands are equal or not, if yes then condition becomes true (=), Checks if the values of two operands are equal or not, if values are not equal then condition becomes true (!=).


  • Relational Operator:

    • Compare range of value. (>)Greater than,(<) Less than,(>=) Greater than equal to,(<=) Less than equal to.


  • Logical Operator:

  • AND Operator:- If all the condition are satisfy then we use AND Operator.


  • OR Operator:- If any one condition are satisfied we get result.


  • NOT Operator:- Logical inverse operator.


  • Special Operator:

  • IN Operator:- It is multivalue operator which accept multiple values at the RHS.

    •                        Syntax:-Column_Name / Expression IN (V1,V2,V3.....Vn)


  • NOT IN Operator:- It is similar to IN operator instead of selecting the values, values are getting rejected.

    •                        Syntax:-Column_Name / Expression NOT IN (V1,V2,V3.....Vn)


  • Between Operator:- It is used to select the range of values.

    •                        Syntax:-Column_Name between Lower range (Value) and Higher range (Value)


  • NOT Betweeen Operator:- It is similar to Between operator instead of selecting the values, values are getting rejected.

    •                        Syntax:-Column_Name between Lower range (Value) and Higher range (Value)


  • IS Operator:- It is used to compare with the null value.

    •                        Syntax:-Column_Name is null;


  • IS NOT Operator:- It is used to compare not null value.

    •                        Syntax:-Column_Name IS NOT null;


  • Like Operator:- It is used for "Pattern to match".

    •                        Syntax:-Column_Name between Lower range (Value) and Higher range (Value)


  • NOT Like Operator:- It is similar to Like operator instead of selecting the values, values are getting rejected.

    •                        Syntax:-Column_Name NOT Like "Pattern to match".


  • Subquery Operator:

  • All Operator:- All operator returns true, if all values at R.H.S. satisfy the condition.


  • Any Operator:- Any operator returns true, if any one of the values at R.H.S. satisfy the condition.


  • Exits Operator:-If subquery return's any value other than null we call it as a Exists Operator.


  • NOT Exits Operator:- Not Exits operator return's true, if subquery return's null value.

    • JOINS

    It is used to retrive the data from multiple table simultaneously is called Joins.


    Why We Use Joins?

    Whenever the attributes to be selected from both table we use join.


  • Types Of Joins:
  • Cartesian Join / Cross Join:

    • In Cartesian join table 1 record are merged with all the record of table 2.

                           Oracle Syntax:-
                                           select columan _name
                                           from table_name 1 , table _name 2;


  • Inner Join / Equi Join:

    • It is used to find only matching records or records which has Pairs.


  • Join Condition: The condition on which two tables are merged.

    •                            Table_Name 1.Column_Name = Table_Name 2.Column_Name

                           Oracle Syntax:-
                                           select columan _name
                                           table_name 1 ,table _name 2
                                           Where < Join_Condition >;


  • Self Join:

    • Joining the table by itself is known as self join.


  • Join Condition: The condition on which two tables are merged.

    •                            Column_Name.Table_Name 1 = Column_Name.Table_Name 2
                               E1.MGR = E2.EMP

                           Oracle Syntax:-
                                           select columan _name
                                           table_name 1 ,table _name 2
                                           Where < Join_Condition >;


  • Outer Join:

    • It is used to obtained unmatched records.


  • Type Of Outer Join:
  • Left Outer Join:

    • It is used to obtain un-matched record of left table along with matching records.

                           Oracle Syntax:-
                                           columan _name
                                           table_name 1 ,table _name 2
                                           T1.Column_Name = T2.Column_Name(+);


  • Right Outer Join:

    • It is used to obtain un-matched record of right table along with matching records.

                           Oracle Syntax:-
                                           columan _name
                                           table_name 1 ,table _name 2
                                           T1.Column_Name(+) = T2.Column_Name;


  • Full Outer Join:

    • It is used to obtain un-matched record both left and right table along with matching records.

                           ANSI Syntax:-
                                           columan _name
                                           table_name 1 full(outer)Join table _name 2
                                           Join_Condition >;


    CORRELATED SUBQUERY

    A Query return inside another query such that outer query and inner query are inter-dependent on each other.


  • Working Priciple:


  • Let use consider two queries outer query and inner query.
  • In correlated subquery outer query will start the execution and produces the partial output.
  • Whatever the output produces by the outer query it will taken as an input as an inner query.
  • Whatever the output produces by the inner query,It will taken as on input to the outer query.
  • Finally outer query will process it and produces the result.
  • Inner query and outer query are inter-dependent on each other.
