DDL(DATA DEFINITION LANGUAGE)

DDL or Data Definition Language actually consists of the SQL commands that can be used to define the database schema. It simply deals with descriptions of the database schema and is used to create and modify the structure of database objects in the database.DDL is a set of SQL commands used to create, modify, and delete database structures but not data. These commands are normally not used by a general user, who should be accessing the database via an application.


  • List of DDL commands:
  • CREATE:

    • This command is used to create the database or its objects (like table, index, function, views, store procedure, and triggers)


  • ALTER:

    • This is used to alter the structure of the database.


  • DROP:

    • This command is used to delete objects from the database.


  • TRUNCATE:

    • This is used to remove all records from a table, including all spaces allocated for the records are removed.


  • COMMENT:

    • This is used to add comments to the data dictionary.


  • RENAME:

    • This is used to rename an object existing in the database.

    DQL (DATA QUERY LANGUAGE)

    DQL statements are used for performing queries on the data within schema objects. The purpose of the DQL Command is to get some schema relation based on the query passed to it. We can define DQL as follows it is a component of SQL statement that allows getting data from the database and imposing order upon it. It includes the SELECT statement. This command allows getting the data out of the database to perform operations with it. When a SELECT is fired against a table or tables the result is compiled into a further temporary table, which is displayed or perhaps received by the program i.e. a front-end.


  • List of DQL commands:
  • SELECT:

    • It is used to retrieve data from the database.


  • PROJECTION:

    • This is used to retrive the data from columns.


  • SELECTION:

    • This is used to retrive the data from both columns and rows.


  • JOINS:

    • This is used to retrive the data from multiple Tables simultaneously.

    DML (DATA MANIPULATION LANGUAGE)

    The SQL commands that deals with the manipulation of data present in the database belong to DML or Data Manipulation Language and this includes most of the SQL statements. It is the component of the SQL statement that controls access to data and to the database. Basically, DCL statements are grouped with DML statements.


  • List of DML commands:
  • INSERT:

    • It is used to insert data into a table.


  • UPDATE:

    • It is used to update existing data within a table.


  • DELETE:

    • It is used to delete records from a database table.


  • LOCK:

    • Table control concurrency.

    DCL (DATA CONTROL LANGUAGE)

    TDCL includes commands such as GRANT and REVOKE which mainly deal with the rights, permissions, and other controls of the database system.<


  • List of DCL commands:
  • GRANT:

    • This command gives users access privileges to the database.


  • REVOKE:

    • This command withdraws the user’s access privileges given by using the GRANT command.

    TCL (TRANSACTIONAL CONTROL LANGUAGE)

    TCL stands for Transaction Control Languages. These commands are used for maintaining consistency of the database and for the management of transactions made by the DML commands.


  • List of TCL commands:
  • COMMIT:

    • Commits a Transaction.


  • ROLLBACK:

    • Rollbacks a transaction in case of any error occurs.


  • SAVEPOINT:

    • Sets a savepoint within a transaction.

    NORMALIZATION

    Process of reducing larger table into smaller table by identifying redundancy and anomalies is known as normalization.


  • Redundancies:

    • Repeatation of data or duplication of data.


  • Anomalies:

    • They can be termed as a side effects that we can have in the form of Insert, Update, Delete.


  • Types of Normalization:
  • 1NF:

    • Data entered in a cell should accept single value data.If table is unconditional, it converts into "TOTAL FUNCNTIONAL DEPENDANCIES".


  • 2NF:

    • If the table is under 2NF first it should complete 1NF. If the table is having "TRANSITIVE FUNCTIONAL DEPENDANCY" it should convert into "TOTAL FUNCTIONAL DEPENDANCY".


  • 3NF:

    • If the table is under 3NF first it should complete 2NF. If the table is having "PARTIAL FUNCTIONAL DEPENDANCY" it should convert into "TOTAL FUNCTIONAL DEPENDANCY".


  • Types of Dependancies:
  • Total Functional Dependancies:

    • All the attributes depends on one attribute.


  • Partial Functional Dependancies:

    • Composite key attribute depends on 2 attributes.


  • Transitive Functional Dependancies:

    • Attribute indirectly depends on another attribute.One non key attribute depends on key attribute.

    DE-NORMALIZATION

    It is a process of combing 1 smaller table to form a bigger table it is known as DE-NORMALIZATION.

    ATTRIBUTES

  • Types of Attributes:
  • Key Attribute:

    • It is also known as candidatee key.It is an attribute present in the table.


  • Non-Key Attribute:

    • Non-key attributes are not unique.


  • Prime-Key Attribute:

    • key's which are eligible to become a primary key but among only one can be selected as a prime key attribute.It is an attribute present in the table.


  • Super-Key Attribute:

    • Combination of 2 or more key attributes.


  • Composite-Key Attribute:

    • Combination of 2 or more non key attributes.


  • Foreign-Key Attribute:

    • It is used to have connection between 2 tables.

    E.R.DIAGRAM

  • It's Components:
  • ENTITY:

    • Specify the condition in rectangle.


  • ATTRIBUTES:

    • Specify the condition in oval.


  • RELATIONSHIP:

    • Specify the condition in hexagon.

    PSEUDO COLUMN

    A pseudo column behaves like table column, but it is not actually stored in column.


  • It's Components:
  • ROWNUM:

    • Oracle engine maintain number of records inserted by user in table.By the help of Rownum clause we can access the data accordingly to records inserted.


  • ROW-ID:

    • For each row in database row-id pseudo column returns row's address. The row-id contains information about rows address.


  • DATA BLOCK NO:

    • The space assigned by the oracle SQL engine to save the record.


  • RECORD NO:

    • Oracle engine maintains the record number for each record.


  • FILE NO:

    • File no means table number.
