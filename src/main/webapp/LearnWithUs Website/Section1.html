    SUBQUERY

    A Query Return inside anather Query we call it as a subquery.


  • Working Priciple:
  • Always in subquery inner Query / subquery will execute first & Produce output.
  • Whatever the output produce by inner Query it will be taken as input to outer Query.
  • Outer Query will process it and produces the Result
  • Therefore we can say that outer query is totally dependant Inner Query.

  • why & when we use subquery?
  • Case 1:

    • whenever there is unknown Present in question to find that Unknown we use Subquery.

  • Case 2:

    • Whenever data to be selected and condition are executed are present in differnt table then we use subquery.


  • TYPES OF SUBQUERY
  • Single row subquery:

    • If the subquery Returns exactly one value we call it as a Single row Subquery.

    If Subquery Returns exactly one value we have use either normal operator (=) or Special Operator (IN, NOT IN, IS, IS NOT, BETWEEN, NOT BETWEEN).


  • Multi row subquery:

    • If Subquery returns more than one value we call it as Multi Row Subquery.

    If subquery Returns more than one value we have to use Special operator Always.


  • NOTE:
  • It is very difficult to identify whether query belong to single Row Subquery or multi Row Subquery it is Recommanded that always use Special operator in subquery instead of Normal operater.

    • Nested Subquery

    A subquery written inside another subquery is known as nested subquery.
